# V2.1 美术资产规范与方向

> 阶段：V2.1 —— 从"贴图拼页面"升级为"真正像素经营游戏场景"
> 日期：2026-05-20
> 前置版本：V2.0-alpha (Phaser 办公室场景壳)

## 一、当前审计结论

### 1.1 可继续使用的资产

| 资产 | 状态 | 原因 |
|------|------|------|
| `player-idle` / `player-working` / `player-tired` | 临时可用 | 有透明背景，但非 spritesheet，无方向帧 |
| `alert-lamp` | 可用 | 反馈图标，不需要 tileset 化 |
| `pass-stamp` / `fail-stamp` | 可用 | 单帧效果，后续可升级为 sheet |
| `coin-reward` / `reputation-star-badge` | 可用 | HUD 图标，尺寸小，不需要 tileset 化 |
| `folder-accounting/tax/law` | 可用 | CaseBoard DOM 中使用，非 Phaser 场景 |

### 1.2 应该废弃或重做的资产

| 资产 | 问题 | 处理方式 |
|------|------|---------|
| `desk-computer` (256) | 单张立绘，无统一透视，像插画而非游戏物件 | 废弃，重做 3/4 top-down workstation |
| `case-board` (256) | 同上，视角不统一 | 废弃，重做 wall-mounted board |
| `filing-cabinet` (256) | 视角偏正面，不兼容 top-down | 废弃，重做 3/4 cabinet |
| `printer` / `coffee-cup` / `wall-clock` | 单张独立，无阴影，像贴纸 | 废弃，统一透视重做 |
| `window-sunny` | 有白边，不兼容暗色背景 | 废弃，重做透明背景 window |
| `wood-floor-tile` / `wall-panel` | 非 seamless，有接缝 | 废弃，重做 seamless tileset |

### 1.3 只能临时使用的资产

| 资产 | 临时期限 |
|------|---------|
| 所有 v2 256 角色图 | V2.2 前必须替换为 spritesheet |
| 所有 v2 128 反馈图标 | V2.3 前可升级动画 sheet，但当前可用 |
| 当前 `file-stack` | V2.2 前重做，当前先显示数量状态 |

### 1.4 当前场景最影响质感的问题

1. **无 tileset**：地板用 Graphics 绘制线条，墙面纯色填充，无真实纹理
2. **透视混乱**：物件有的偏正面、有的偏侧面、有的像立绘，不在同一空间
3. **无阴影**：所有物件悬空，没有接地阴影
4. **主角无方向**：只有正面 idle，不能走动、不能转向
5. **比例失调**：咖啡杯和文件柜几乎一样大
6. **交互反馈弱**：hover 只有缩放，没有高亮框、没有文字提示
7. **像网页贴图**：因为以上所有问题叠加，用户感知是"图片堆在网页上"而非"在游戏里"

## 二、美术目标

### 2.1 参考方向

- **温暖、清晰、低干扰的像素风事务所**
- 不是暗黑紫色网页 UI（当前 CSS 变量需要调整）
- 不是卡片堆叠（当前 CaseBoard 是 DOM 卡片，保留但场景内不用卡片）
- 更像一个**可经营的小办公室**
- **学习工具属性仍然优先**，不能过度花哨

### 2.2 情绪板关键词

- Stardew Valley 办公室 mod
- Papers, Please 的严肃但不压抑
- 开罗游戏 (Kairosoft) 的清晰可读
- 日本像素 RPG 的温馨日常感

## 三、统一视角

### 3.1 技术规格

| 项目 | 规格 |
|------|------|
| 视角 | 3/4 top-down (isometric-ish but not true iso) |
| 投影 | Orthographic |
| Tile grid | 32px |
| 角色尺寸 | 32×48 px (1×1.5 tiles) |
| 家具尺寸 | 按 1×1, 2×1, 2×2 tiles 设计 |
| 阴影 | 统一 45° 投射，半透明黑色椭圆 |

### 3.2 视角示意图

```
      ↑ 北（上方）
      │
  ←───┼───→  玩家看到角色的头顶+正面+右侧面
      │
      ↓ 南（下方）

角色站立：能看到头顶、脸、右肩
家具摆放：顶面+正面+右侧面可见
```

## 四、色彩规范

### 4.1 主色板

| 用途 | 色值 | 说明 |
|------|------|------|
| 墙面 | `#8B9A7C` | 灰绿色，温暖不刺眼 |
| 地板 | `#C4A77D` | 浅木色，比当前紫色温暖 |
| 地板暗纹 | `#B89B6F` | 木纹变化 |
| 家具木色 | `#8B6914` | 中等胡桃木 |
| 金属/电脑 | `#5A6B7C` | 冷灰蓝，与暖木对比 |
| 高亮交互 | `#E8A838` | 琥珀金，比当前青色更温暖 |
| 风险警告 | `#D94F4F` | 柔和红，不刺眼 |
| 成功/通过 | `#5AAD5A` | 柔和绿 |
| 会计科目 | `#4A9B8E` | 青绿，沉稳 |
| 税法科目 | `#C4943A` | 金黄，对应"金税" |
| 经济法科目 | `#7A6B9E` | 柔和紫，不跳 |

### 4.2 禁止

- ❌ 大面积 `#1a1c2e` 紫黑背景
- ❌ 纯白背景 `#FFFFFF`
- ❌ 高饱和 `#FF0000` / `#00FF00`
- ❌ 渐变填充（保持像素 flat color）

### 4.3 文字可读性

- 所有文字区域必须有半透明深色底 `#000000` @ 60% opacity
- 文字颜色 `#F0E6D3`（暖白）
- 最小字号 12px（等效 16px 屏幕像素）

## 六、命名规范

### 6.1 规则

- ✅ 全小写 `lowercase`
- ✅ `snake_case`（推荐）或 `kebab-case`（二选一，项目统一 snake_case）
- ✅ 文件名表达尺寸和用途：`floor_wood_32.png`, `player_walk_down_sheet.png`
- ❌ 禁止 UUID：`a1b2c3d4.png`
- ❌ 禁止中文：`玩家.png`
- ❌ 禁止空格：`player idle.png`
- ❌ 禁止无意义编号：`image1.png`, `asset_01.png`

### 6.2 示例

```
✅ floor_wood_32.png
✅ desk_workstation_2x2.png
✅ player_idle_down.png
✅ pass_stamp_effect_sheet.png

❌ wood floor.png
❌ desk2.png
❌ player1.png
❌ 12345678.png
```

## 七、透明背景要求

### 7.1 必须透明

- 所有 props / character / effects / UI skin
- 格式：PNG-24 with alpha channel
- 禁止：白底、黑底、灰底、边缘半透明白框

### 7.2 tileset 特殊要求

- seamless：四边可无缝拼接
- 禁止：边缘有渐变或阴影（接缝会暴露）
- 测试：2×2 平铺后无可见接缝

### 7.3 spritesheet 特殊要求

- 每帧尺寸完全一致
- 帧间距为 0（紧密排列）
- 禁止：每帧不同尺寸、帧间有分割线

## 八、当前代码问题与 V2.1 改造范围

### 8.1 不改的（红线）

- `src/store/game-store.ts`
- `src/engine/*`
- `src/types/game.ts`
- `src/components/BattleView.tsx`（逻辑）
- `src/components/ReviewPanel.tsx`（逻辑）
- 存档格式

### 8.2 可改的（低风险）

- `src/phaser/scenes/OfficeScene.ts` — 场景布局、对象创建
- `src/phaser/utils/asset-loader.ts` — 资产路径更新
- `src/phaser/GameConfig.ts` — 分辨率、背景色
- `src/components/GameScene.tsx` — bridge 逻辑可微调
- `src/App.css` — 颜色变量更新

### 8.3 推荐实施顺序

1. **Batch 1**：生成 tileset（地板/墙面/阴影）→ 替换 `createBackground()` 的 Graphics 为 tilemap
2. **Batch 2**：生成 props（办公桌/案件板/文件柜等）→ 替换场景中所有装饰物件
3. **Batch 3**：生成主角 spritesheet → 替换单张角色图为动画精灵
4. **Batch 4**：生成 feedback effects → 替换 BattleView 中的静态反馈
5. **Batch 5**：UI skin → 更新 ResourceBar / CaseBoard 边框样式
6. **代码整理**：抽常量、加注释、tile grid 布局
