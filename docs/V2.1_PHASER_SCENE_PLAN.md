# V2.1 Phaser 场景改造方案

> 目标：从自由坐标摆放升级为 tile grid 布局
> 不改业务逻辑，不改 store/engine/types
> 暂时不引入 Tiled 编辑器

---

## 一、当前问题

### 1.1 自由坐标的问题

```typescript
// 当前 OfficeScene.ts —— 硬编码像素坐标
this.desk = this.add.image(GAME_WIDTH / 2, 480, 'desk').setScale(0.45);
this.caseBoard = this.add.image(GAME_WIDTH / 2, 200, 'case-board').setScale(0.5);
this.player = this.add.image(GAME_WIDTH / 2, 580, 'player-idle').setScale(0.25);
```

问题：
- 坐标是魔法数字，无法快速调整布局
- 物件之间没有空间关系（ desk 和 player 的距离是硬编码的）
- 无法支持"房间扩建"（加宽办公室需要改所有坐标）
- 不同分辨率下比例失调

### 1.2 背景绘制的问题

```typescript
// 当前 —— 用 Graphics 画线条模拟地板
graphics.fillStyle(0x5b606f, 1);
graphics.fillRect(0, floorTop, GAME_WIDTH, GAME_HEIGHT - floorTop);
graphics.lineStyle(2, 0x4a3124, 0.45);
// ... 大量线条绘制
```

问题：
- 不是真正的 tileset，无法替换纹理
- 每帧重绘（虽然 Phaser 会缓存，但不优雅）
- 无法做地板变体（不同区域不同地板）

## 二、改造方案：TypeScript 数组描述地图

### 2.1 核心思想

不用 Tiled 编辑器，用 TypeScript 二维数组描述地图，Phaser 读取数组生成 tilemap。


### 2.2 优势

| 优势 | 说明 |
|------|------|
| 纯代码 | 不需要 Tiled 编辑器，不增加构建步骤 |
| 类型安全 | TypeScript 检查，不会写错 tile 类型 |
| 版本控制友好 | 数组 diff 比二进制 .tmx 文件清晰 |
| 运行时生成 | 可支持随机地图、程序生成房间 |
| 容易扩展 | 加一行数组就能扩建房间 |

```typescript
export function preloadAllAssets(scene: Phaser.Scene) {
  // tileset
  Object.entries(TILESET_ASSETS).forEach(([key, path]) => scene.load.image(key, path));
  
  // props
  Object.entries(PROP_ASSETS).forEach(([key, path]) => scene.load.image(key, path));
  
  // legacy (临时)
  Object.entries(LEGACY_ASSETS).forEach(([key, path]) => scene.load.image(key, path));
  
  // character sheets
  Object.entries(CHARACTER_SHEETS).forEach(([key, config]) => {
    if (typeof config === 'string') {
      scene.load.image(key, config);
    } else {
      scene.load.spritesheet(key, config.path, {
        frameWidth: config.frameWidth,
        frameHeight: config.frameHeight,
      });
    }
  });
  
  // effect sheets
  Object.entries(EFFECT_SHEETS).forEach(([key, config]) => {
    scene.load.spritesheet(key, config.path, {
      frameWidth: config.frameWidth,
      frameHeight: config.frameHeight,
    });
  });
}
```

## 四、OfficeScene 改造步骤

### 4.1 步骤 1：抽常量（Task 5 已完成）

```typescript
// src/phaser/scenes/OfficeScene.ts
import { TILE_SIZE, MAP_WIDTH_TILES, MAP_HEIGHT_TILES } from '../map/office-map';

export class OfficeScene extends Phaser.Scene {
  // 不再硬编码坐标，所有位置通过 tileToPixel() 转换
  private tileToPixel(tx: number, ty: number): { x: number; y: number } {
    return {
      x: tx * TILE_SIZE + TILE_SIZE / 2,
      y: ty * TILE_SIZE + TILE_SIZE / 2,
    };
  }
}
```

### 4.2 步骤 2：tilemap 背景

```typescript
private createTilemapBackground() {
  // 创建空白 tilemap
  const map = this.make.tilemap({
    tileWidth: TILE_SIZE,
    tileHeight: TILE_SIZE,
    width: MAP_WIDTH_TILES,
    height: MAP_HEIGHT_TILES,
  });
  
  // 添加 tileset
  const floorTiles = map.addTilesetImage('floor_wood', 'floor_wood');
  const wallTiles = map.addTilesetImage('wall_plain', 'wall_plain');
  
  // 创建图层
  const floorLayer = map.createBlankLayer('floor', floorTiles);
  const wallLayer = map.createBlankLayer('wall', wallTiles);
  
  // 从数组填充
  FLOOR_LAYER.forEach((row, y) => {
    row.forEach((tileId, x) => {
      if (tileId !== 0) floorLayer.putTileAt(tileId, x, y);
    });
  });
  
  WALL_LAYER.forEach((row, y) => {
    row.forEach((tileType, x) => {
      if (tileType !== '0') wallLayer.putTileAt(this.wallCharToId(tileType), x, y);
    });
  });
}
```

### 4.3 步骤 3：物件按数组摆放

```typescript
private createObjectsFromMap() {
  OBJECTS.forEach(obj => {
    const pos = this.tileToPixel(obj.x, obj.y);
    const sprite = this.add.image(pos.x, pos.y, obj.type);
    
    // 添加阴影
    const shadowPos = this.tileToPixel(obj.x, obj.y + 0.3);
    const shadow = this.add.image(shadowPos.x, shadowPos.y, 'shadow_blob');
    shadow.setAlpha(0.3);
    shadow.setDepth(sprite.depth - 1);
    
    if (obj.interactive) {
      sprite.setInteractive({ cursor: 'pointer' });
      sprite.on('pointerdown', () => {
        this.game.events.emit(obj.action);
      });
    }
  });
}
```

```typescript
private updateDynamicObjects() {

  // 根据状态显示/隐藏 alert_lamp

  }
}
```

## 八、风险点

| 风险 | 缓解 |
|------|------|
| tileset 生成失败（AI 出不了 seamless）| 先用纯色 tile 占位，后续再换 |
| spritesheet 帧不对齐 | 生成后手动检查，不对齐则单帧加载 |
| 包体积进一步增大 | 只加载当前场景需要的资产，其他动态 import |
| 性能下降（tilemap + sprite 同屏）| Phaser 自动 culling，无需担心 |
| 存档兼容 | 完全不改 save-load，零风险 |
